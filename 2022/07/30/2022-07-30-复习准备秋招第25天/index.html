P52

ROLLBACK 只能针对未COMMIT的数据

常考题：TRUNCATE TABLE xxx 和 DELETE FROM xxx 有啥区别

  • 相同点：
    • 都可以清除表数据，保留表结构
  • 不同点
    • 是否可以回滚
      • TRUNCATE TABLE 不可以
      • DELETE TABLE 看加不加参数
    • 是否可以不全清除
      • TRUNCATE TABLE 只能全部清除
      • DELETE TABLE 可以用 WHERE 条件选择性清除
    • 速度比较
      • TRUNCATE TABLE 更快，因为占用系统和事务日志的资源少

常考题：DDL（Data Definition Language） 和 DML（Data Manipulation Language） 有啥区别

  • 是否可以回滚
    • DDL：一定不能回滚，每次执行完自动COMMIT，不受 SET autocommit = FALSE影响
    • DML：默认不可回滚，但可以加 SET autocommit = FALSE来允许回滚

P61

浮点数类型，即使声明成无符号，范围也没有扩大

MySQL支持非标准语法：FLOAT(a,b)，代表一共a位，其中小数部分b位

  • 例如设置FLOAT(5,2)
    • 添加123.45，没啥问题
    • 添加123.456，自动四舍五入成 123.46
    • 添加999.995，报错
    • 添加1999.99，报错

注意，跟其他类型的语言一样，浮点类型判断相等不用 =

DECIMAL：同样有精度M和标度D的概念，占用M+2个字节，如果不指定，默认是 DECIMAL(10, 0)

  • DECIMAL精度之所以高，是因为底层是字符串存储的

浮点数和定点数的比较

  • 表示范围：相同空间，浮点数能表示更大范围的数
  • 精度：定点数精度更高（任何涉及到金钱的项目）

DATETIME 和 TIMESTAMP 的区别

  • 存储大小：8字节，4字节
  • 底层存储：前者就是存的时间，后者底层存的是毫秒数
  • 关于时区：因为底层存储不同，后者在真正显示的时候，会减去当前时区再显示，所以数据库中的时间，如果切换时间，前者显示不变，后者会变
  • 检索比较速度：后者存的是毫秒数，所以运算会更快
