P21

记录型信号量的具体过程

  • P操作

    • 先S.val–
    • 再判断是否S.val<0，如果是，则进行将自己block，即把自己放到S.L队列里

  • V操作

    • 先S.val++
    • 再判断是否S.val<=0，如果是，则进行wakeup，即唤醒S.L队列里的一个进程

  • 作用

    • 实现系统资源的申请和释放
    • 实现进程的互斥和同步

P22

信号量实现进程的互斥和同步

  • 互斥
    • 临界区前面加P，临界区后面家V
    • 有多少个资源，就把mutex初始值设置为多少
  • 同步
    • 设程序A执行完后才能执行B，则 A -> V -> P -> B
    • 如果是多个程序又相互依赖关系，那么没有一条有向边，就要定义一条mutex

P23

生产者消费者问题

  • 互斥的P操作一定在同步的V操作之后，不然会死锁
  • V操作不会导致进程阻塞，所以互斥和同步的V可以随便交换
  • 生产和消费的过程，不建议放在PV之间，因为会导致临界区代码量变大，并发度降低（即锁住的概率会变大）

P24

多生产者多消费者问题

  • 互斥同步关系
    • 互斥：对盘子的访问
    • 同步：
      • 父亲 -> V -> P -> 女儿：apple
      • 母亲 -> V -> P -> 儿子：orange
      • 儿子女儿 -> V -> P -> 父亲母亲：plate
  • 这道题中，因为任何时刻，三个同步量只有一个为1，因此对于这种缓冲区也只有一个的情况下，mutex可以省去
    • 注意，并不是所有缓冲区为1都可以省去mutex，要确认多个同步量只有一个为1类似的条件

P26

读者写者问题

  • 这里面就是单纯的互斥问题
    • 普通的互斥量rw，不能让读读不互斥
    • 加入count变量，记录当前正在读的进程数量，count>0时，跳过P(rw)，这样表面就能解决读读互斥的问题
    • 但是因为检查count和让count++，不是原子性完成，因此额外加一个mutex来保证检查和赋值的原子性
    • 上述方法会导致写饥饿，因为源源不断的读会让写饿死，因此要额外加入w变量，实现先来先到的原则，即“读写公平法”。

P30

预防死锁

  • 资源互斥
    • 用SPOOLing技术，将互斥资源改成共享资源（其实是缓冲区暂存了请求，这样看起来就不会阻塞）
    • 缺点：有些资源是不能被改成共享的
  • 不可剥夺
    • 要么请求的时候立即剥夺，要么再加上优先级高才能剥夺
    • 缺点：只适用于容易保存和恢复的资源，如CPU；反复申请和释放，降低系统吞吐量，效率变低
  • 请求和保持
    • 静态分配方法：一次性申请完所有资源，这样处理的时候不会再请求
      • 缺点：资源利用率；有可能有线程饥饿
  • 循环等待
    • 顺序资源分配法：每个进程必须按照编号递增的顺序请求资源，编号相同的资源一次性申请完
    • 缺点
      • 不方便新加设备，要重新编号
      • 进程实际使用顺序和编号顺序可能不一致，会资源浪费
      • 按次序申请资源，编程麻烦

P33

程序编写到执行的流程

程序编写到执行的流程

  • .c 文件
    • 编译：高级语言转成机器语言（.c -> .o）
  • .o 文件
    • 链接：目标模块(.o )合成一个大的模块文件（.o -> .exe），这一步形成完整逻辑地址
  • .exe 文件
    • 装入（装载）：.exe文件装到内存里运行，这一步形成完整物理地址
  • 内存中的程序段

装入的三种方式

  • 绝对装入
    • 例如，单道程序，编译时就能产生最终地址
  • 静态重定位 / 可重定位装入
    • 装入的时候，一次性算出最终地址，不能改变
  • 动态重定位 / 动态运行时装入
    • 中间有重定位寄存器可以动态调整位置，运行时才有最终地址

P39

页表项长度，页表长度，页面大小

页表 -> 页表项 == 进程 -> 页面

4GB/2^32B内存，页面大小4KB/2^12B

  • 一共有2^20个页面
  • 因此页面编号要20位，也就是2~3个B来存储一个页面index
  • 因此页表里的每一个页表项是3B
  • 页表长度M：指的是PCB中，或者运行时PCB恢复到页表寄存器中的第二个变量M（页表在内存中的起始地址F + 页表长度M），即该进程一共有几个页（例如进程占用16KB，一个页是1KB，那么页表长度M就是16，即一共有16个页 ），或者说进程所代表的页表，拥有的页表项数量（假设一个页表项是3B，页表48B，那么页表长度M就是16，即页表项有16个）
  • 页表项长度：即每个页表项占用了多少空间，在上面的例子里，就是3B
  • 页面大小：即一个页占用的空间，这个是由系统确定的，比如上面的例子就是1KB
