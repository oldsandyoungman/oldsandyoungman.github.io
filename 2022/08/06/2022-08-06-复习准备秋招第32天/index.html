操作系统

来源：王道计算机考研 操作系统

P7

系统调用的概念

  • 为统一管理，防止混乱，系统的所有共享资源都操作系统统一管理，所有资源相关操作，

系统调用的背后过程

  • 传参，将参数放到一些通用寄存器里：用户态
  • 执行陷入指令（trap指令，访管指令），发出一个内中断，使CPU进入核心态：用户态
  • 执行内核程序处理系统调用：内核态
  • 返回

P8

进程，进程实体的概念

  • PCB
    • 进程存在的唯一标志（存放类似进程的必要信息）
  • 进程实体（静态）
    • PCB + 程序段 + 数据段
  • 进程（动态）
    • 进程是进程实体的运行过程
    • 进程是系统进行资源分配和调度（引入线程后就不再是调度的基本单位了）的一个独立单位

进程的组织方式

按照进程状态（运行，就绪，阻塞）

  • 链接方式
    • 分为多个队列，操作系统由各个队列的头指针
  • 索引方式
    • 分为多个索引表，操作系统由各个索引表的地址

进程的5大特征

  • 动态性（最基本特征）
    • 程序的一次执行过程
  • 并发性
    • 多个进程可以并发执行
  • 独立性
    • 资源、调度的基本单位
  • 异步性
    • 各个进程独立进行，速度不一地前进，结果可能不确定
    • 靠同步机制来解决异步问题
  • 结构性
    • 每个进程都有一个PCB
    • 进程/进程实体：PCB + 程序段 + 数据段

P9

进程的不同状态

  • 运行态：有CPU的资源
  • 就绪态：有除CPU的其他所有资源
  • 阻塞态：除CPU资源外，还缺别的资源
  • 创建态：操作系统给进程分配资源，初始化PCB
  • 终止态：操作系统撤销进程，从中回收资源，撤销PCB

进程之间的转换，关于主动 OR 被动

  • 运行态 -> 阻塞态：主动

    • 进程 主动 申请某种系统资源（系统调用）/请求等待某个事件发生

  • 阻塞态 -> 就绪态：被动

    • 之前申请的资源被分配，等待的事件被发生

刷题

来源：labuladong的算法小抄

450. Delete Node in a BST

  • 如果要删除的节点左右子树都非空

    • 找到右子树最小节点，记为 replace_node

    • root.right = dfs(root.right, replace_node.val)

      • 这步很妙，因为你有指针指向这个地方，所以不用怕

    • 将 replace_node 左右节点接到原来 root 的左右子树位置

      • 注意，root左右指针是否置空，其实不重要

98. Validate Binary Search Tree

  • 就从上到下也行，dfs的形参定义是：对于root为根节点的子树，是否能保证在min 和 max 的范围内
  • 注意别漏掉等号

96. Unique Binary Search Trees

  • 边界条件，是返回1，不是0
    • 同理，另外一个题，返回的是ArrayList里面有个null，而不是空的ArrayList
  • 注意这道题要用备忘录
    • 另外一个完整输出的题不需要备忘录
