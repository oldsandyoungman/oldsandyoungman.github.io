P4~6

耦合度的解释
  • 如果A类内部方法调用了B类，用直接的new的方式，之后无论是是换一个新的B+类，还是改B类中的方法名，都得同步改A中的方法， 这样只是改B二被迫同时改A，成本太高了
工厂模式的本质
  • 解决耦合度太高的问题（当然IOC解决得更彻底）
IOC的本质
  • 框架 = 注解/xml + 反射 + 设计模式（工厂模式）
IOC容器的定义，两种实现方式+区别
  • 定义：底层就是对象工厂
  • 实现方式：
    • BeanFactory：
      • Spring内部的接口，开发一般不使用
      • 加载配置文件的时候不会创建对象，获取（使用）对象时才会创建
    • ApplicationContext：
      • BeanFactory 的子接口，功能更多
      • 加载配置文件的时候直接创建对象
        • 这种方法更好，更适用于服务器，因为使用过程中的慢是不能接受的，但启动的时候无所谓

Bean管理的内容

    • java 代码里

  • 注入属性

    • 类的 set 方法

      • 类中定义 set 方法

      • xml 中创建对象

      • java中调用方法

    • 有参构造

      • 类中定义有参构造方法

      • xml 中创建对象

      • java中调用方法
